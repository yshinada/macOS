GitHub Pagesとは

Q. GitHub Pages とは

GitHub Pages は、GitHub に置いた index.html などのファイルを、そのまま無料でWebサイトとして公開できる機能です。

今回作った index.html、style.css、script.js、assets/hero-software-workspace.png を GitHub にアップロードすると、 https://ユーザー名.github.io/リポジトリ名/ のようなURLで公開できます。

WordPress.com と違って、今回作った HTML / CSS / JavaScript をほぼそのまま使えます。 個人開発ソフトの配布ページには GitHub Pages がかなり相性のよい公開方法です。

手順は、GitHubアカウントを作る、新しいリポジトリを作る、サイトファイルをアップロードする、 Settings → Pages を開く、main ブランチを選んで公開する、表示されたURLにアクセスする、という流れです。

配布ファイルが大きい場合、DMG や EXE は GitHub Releases に置いて、サイト側からそこへリンクするのがおすすめです。

Q. リポジトリの Settings → Pages を開くはどこですか

リポジトリ画面の上部メニューにある Settings を開き、左側メニューの Code and automation の中にある Pages をクリックします。

Pages の画面で Source を Deploy from a branch にし、Branch は main、フォルダは /root を選んで Save を押します。 しばらく待つとページ上部に公開URLが表示されます。

まだリポジトリに index.html などが入っていない場合は、Code に戻って Add file → Upload files から index.html、style.css、script.js、assets フォルダをアップロードし、Commit changes を押してから Pages 設定に戻ります。
